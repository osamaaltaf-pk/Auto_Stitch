Implementation Phases
Phase 1 Left panel — vertical icon rail + sub-panels
app.js · index.css Frontend only ▼
⊞
Replace horizontal tab bar with 52px vertical icon rail
Remove MEDIA / SFX / VOICE / MUSIC horizontal tabs from the left panel header. Replace with a fixed 52px-wide vertical column flush to the left edge. Each tab is a 36×36px icon button (icon + 8px label below). SFX and Music tabs removed entirely — they live on timeline track lanes where they belong.
Tabs: Media Effects Filters Text Stickers Transitions
▤
Sub-panel renders to the right of the rail (~200px wide)
Active tab drives which sub-panel renders inside the same left panel slot. The activeLeftTab state variable controls this. Media sub-panel keeps the existing scan/clip browser. All others are new grid UIs.
⊡
Effects + Filters + Stickers — 2-column thumbnail grids
Each tile: 70px wide, 52px tall, rounded border, colored CSS background representing the effect/filter, name label underneath. Active tile gets purple accent border + glow. Clicking writes to selected block metadata and triggers composer re-render. No real thumbnail images needed — CSS backgrounds only.
grid-template-columns: 1fr 1fr gap: 6px border-radius: 7px
⇄
Transitions sub-panel — single column list
Transitions shown as single-column list items (not grid) since they represent named sequences. Items: None (Cut), Fade, Dissolve, Wipe Left, Wipe Right, Zoom In, Zoom Out, Slide Up. Clicking applies transition to the selected block's transition metadata field.
T
Text sub-panel — two sections stacked
Section 1: Auto-Captions — moves existing global subtitle settings (burn checkbox, style mode, font, color, placement, backdrop) here from the right panel Global Settings. Section 2: Slot Text Overlay — new per-slot text annotation (added in Phase 3 with backend).
Phase 2 Real-time CSS overlay system on the composer preview
app.js · index.css Frontend only ▼
👁
CSS filter applied inline to preview video/image element
When a slot is selected, read its color_grading field and apply the mapped CSS filter string directly as style.filter on the preview <video> or <img> element. Instant — no backend, no API call.
cinematic: contrast(1.15) saturate(0.85) sepia(0.1) cool_blue: hue-rotate(20deg) saturate(1.2) brightness(1.05) warm_gold: sepia(0.3) saturate(1.4) brightness(1.02) vintage: sepia(0.4) contrast(0.9) brightness(0.95) hi_contrast:contrast(1.5) saturate(1.1) cyberpunk: hue-rotate(270deg) saturate(2) contrast(1.1) bleach: contrast(1.2) saturate(0.6) brightness(1.1) bw: grayscale(1) contrast(1.1) teal_orange:saturate(1.4) hue-rotate(-15deg) contrast(1.1) indie_warm: sepia(0.2) saturate(1.3) brightness(1.04) contrast(1.05) retro_90s: sepia(0.45) contrast(1.1) saturate(0.85) brightness(1.03)
◈
Overlay div layers rendered on top of the preview for video effects
A container div wraps the preview media element with position:relative. When a slot has an overlay_effect set, an absolutely positioned child div is rendered on top with the matching CSS class. The class drives the animation. Switching slots swaps the class instantly.
★
Sticker overlay — bouncing badge on preview corner
If selected block has a sticker field set, render a small emoji badge absolutely in the top-right corner of the preview frame with a subtle bounce animation. Stickers: 🔥 Fire, ❤️ Like, 🔔 Subscribe, ⚠️ Warning, 💥 Boom, 💡 Idea, 📈 Viral, ⭐ Star.
⟳
Slot selection triggers preview re-render
Existing slot click handler already updates the right panel. Extend it to also call updateComposerOverlays(block) which reads the block's effect/filter/sticker metadata and applies CSS classes and filter styles to the preview element. No new API endpoints.
Phase 3 Text overlay — per-slot annotation with draggable position
app.js · manifest.py · stitcher.py Frontend + Backend ▼
📄
manifest.py — extend VideoBlock with text overlay fields
Add new optional fields to the VideoBlock dataclass. All default to safe values so existing projects load without migration.
text_overlay: str = "" text_overlay_font: str = "arial" text_overlay_size: int = 40 text_overlay_color: str = "white" text_overlay_placement_y: int = 50 # 0–100 % from top text_overlay_box_enabled: bool = False text_overlay_outline_width: int = 3
🎬
stitcher.py — render text_overlay via FFmpeg drawtext
Inside build_ffmpeg_cmd, if block.text_overlay is non-empty, append a drawtext filter with computed y position. Also add new premium color grading filters to the existing color grading map.
drawtext=text='{text}':fontfile='{font}':fontsize={size}: fontcolor={color}:x=(w-text_w)/2: y=h*{placement_y}/100: box={box_enabled}:boxborderw={outline} # New FFmpeg color grading entries: teal_orange: colorbalance=rs=0.12:gs=0.0:bs=-0.1: rm=0.15:gm=-0.03:bm=-0.15 indie_warm: colorbalance=rs=0.1:gs=0.05:bs=-0.05, eq=contrast=1.05:brightness=0.02 retro_90s: eq=contrast=1.1:saturation=0.85:brightness=0.03, colorbalance=rs=0.05:gs=0.02:bs=-0.04
↕
app.js — draggable text block on composer preview
Render the text_overlay string as an absolutely positioned div over the preview. On mousedown, track vertical drag and compute new placement_y as a percentage of preview height. On mouseup, save to block metadata. Cursor changes to ns-resize on hover.
🖊
Text sub-panel — Slot Text Overlay controls
Below the Auto-Captions section (moved from Global Settings), add a Slot Text Overlay section: text input, font dropdown, size slider (12–96), color picker, box/outline toggle, vertical placement slider (0–100%). All inputs sync to selected block metadata in real-time.
Phase 4 Premium media bin — drag & drop + context menu
app.js Frontend only ▼
⊟
Media sub-panel — segmented category tabs
Add a mini tab row inside the Media sub-panel: All · Videos · Voice · SFX · Music. Filters the asset list below. Assets shown as compact cards with filename, type icon, and duration. This is a pure UI filter on the existing asset data — no new API.
⤵
Asset cards draggable → timeline blocks are drop targets
Set draggable=true on asset cards. On dragstart, store asset path via dataTransfer.setData("text/asset", JSON.stringify(asset)). Timeline block lane cells listen for dragover + drop. Valid drop target highlights with a teal glow border. On drop, update block audio/video src.
≡
Right-click context menu on asset cards
On contextmenu event, render an absolute-positioned popup with: Apply to Selected Slot · Insert at End · Delete Asset. Closes on outside click via a transparent overlay div. No third-party library. Pointer-events trap on the menu itself prevents premature close.
